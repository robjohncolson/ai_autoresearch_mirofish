# Summarization Template

When summarizing a repository, produce a markdown document with these sections:

## 1. Purpose
One to three sentences: what does this project do and who is it for?

## 2. Architecture
Mermaid diagram showing major modules and data flow. Keep it high-level
(5-10 nodes max).

## 3. Key Files
Table with columns: Path | Purpose | Lines of Code (approximate)
Include only the most important files (10-15 max).

## 4. Dependencies
External dependencies: packages, APIs, services.

## 5. Recent Activity
Last 5 git commits with dates and one-line summaries.

## 6. Observations
Bullet list of notable patterns, potential issues, or improvement opportunities.
Each item should cite a specific file or pattern.
